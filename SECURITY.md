# Security Policy

## Reporting

Report suspected security or privacy issues through GitHub's private vulnerability reporting for this repository. Do not include production analytics payloads, credentials, personal data, raw device identifiers, or confidential application code in a public issue.

Use public issues for non-sensitive correctness and documentation problems.

## Scope

Relevant reports include instructions that could cause credential exposure, unintended external writes, audit-mode mutation, privacy/consent bypass, sensitive-data collection, identity contamination, fabricated external validation, or unsafe supply-chain behavior.

These skills are instruction packages and do not ship an SDK or analytics service. Application owners remain responsible for validating implementation, platform policy, privacy obligations, and PostHog configuration in their environment.
